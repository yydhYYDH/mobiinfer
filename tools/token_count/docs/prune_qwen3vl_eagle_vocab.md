# Qwen3-VL + Eagle3 Vocabulary Pruning

This flow is intentionally split into two scripts:

1. Prune the target/base Qwen3-VL model vocabulary.
2. Prune the Eagle3 draft vocabulary using the target model's `old_to_new_token_id.json`.

Do not combine the two steps unless you are debugging. The target model is the source of truth for tokenizer id remapping.

## Vocabulary Representation

Target model `/temp/models/Qwen3-VL-2B-Instruct`:

- tokenizer vocab size: `151936`
- embedding tensor: `model.language_model.embed_tokens.weight`
- embedding shape: `[151936, 2048]`

Eagle3 draft `/temp/models/Qwen3-VL-2B-Instruct-Eagle3`:

- target `vocab_size`: `151936`
- `draft_vocab_size`: `32000`
- `lm_head.weight`: `[32000, 2048]`
- `d2t`: `[32000]`
- `t2d`: `[151936]`

Runtime mapping:

```text
target_token_id = draft_token_id + d2t[draft_token_id]
```

After target vocab pruning, every target token id changes. Therefore Eagle3 `lm_head.weight`, `d2t`, `t2d`, and `draft_vocab_size` must be rebuilt from the target model's `old_to_new_token_id.json`.

The Eagle script:

- keeps only draft rows whose mapped old target token survives target pruning
- prunes `lm_head.weight`
- rebuilds `d2t`
- rebuilds `t2d`
- updates `config.json` `vocab_size` and `draft_vocab_size`

## Constraints

- Token ids are tokenizer-specific. Counts from MAI/Qwen2.5/Qwen3.5 are not valid for Qwen3-VL.
- Target vocab pruning without Eagle `d2t/t2d` rebuild will produce incorrect speculative tokens.
- Successful MNN export is not enough. The C++ runtime smoke test is the pass criterion.

## Optional: Prune Eagle Further By Token Counts

After target Qwen3-VL has already been pruned, Eagle can still be smaller than “all draft rows whose target token survived target pruning”. This does not change the target tokenizer or target model. It only removes Eagle draft rows for low-frequency target tokens.

Use `--target-counts` with the same old target token ids used for target pruning. The script filters Eagle draft rows by their mapped old target token id.

Conservative example: keep target tokens with count >= 10 in Eagle, plus special ids from configs:

```bash
ssh sai-a100 '
source /opt/conda/etc/profile.d/conda.sh
conda activate mnn
python /home/ma-user/workspace/csm/mobiinfer/token_count/prune_eagle3_vocab_from_target_mapping.py \
  --eagle-model /temp/models/Qwen3-VL-2B-Instruct-Eagle3 \
  --target-old-to-new /temp/models/Qwen3-VL-2B-Instruct-vocab-pruned/old_to_new_token_id.json \
  --target-config /temp/models/Qwen3-VL-2B-Instruct-vocab-pruned/config.json \
  --target-counts /path/to/qwen3vl_token_counts.csv \
  --min-count 10 \
  --out-eagle /temp/models/Qwen3-VL-2B-Instruct-Eagle3-vocab-pruned-count10 \
  --dry-run
'
```

Aggressive example: keep only the top 16000 counted target tokens that Eagle can map to, plus special ids from configs:

```bash
ssh sai-a100 '
source /opt/conda/etc/profile.d/conda.sh
conda activate mnn
python /home/ma-user/workspace/csm/mobiinfer/token_count/prune_eagle3_vocab_from_target_mapping.py \
  --eagle-model /temp/models/Qwen3-VL-2B-Instruct-Eagle3 \
  --target-old-to-new /temp/models/Qwen3-VL-2B-Instruct-vocab-pruned/old_to_new_token_id.json \
  --target-config /temp/models/Qwen3-VL-2B-Instruct-vocab-pruned/config.json \
  --target-counts /path/to/qwen3vl_token_counts.csv \
  --top-k-targets 16000 \
  --out-eagle /temp/models/Qwen3-VL-2B-Instruct-Eagle3-vocab-pruned-top16000
'
```

If a low-frequency token must remain draftable by Eagle, put its old target id in a text file and pass:

```bash
--keep-target-ids /path/to/force_keep_old_target_ids.txt
```

Recommended workflow:

1. Run `--dry-run` and compare `draft rows surviving target prune` vs `draft rows surviving count filter`.
2. Export MNN.
3. Run the smoke test.
4. Compare acceptance/speed/quality. Over-pruning Eagle can reduce speculative acceptance even if it makes the draft model smaller.
